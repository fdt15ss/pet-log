# AGENTS.md

## 계획 문서 작성 규칙

- 코드 변경이 있으면 완료 전 관련 문서도 함께 업데이트한다.
- 구조, 설정, 실행 방법, provider 경계, 검증 명령이 바뀌면 `README.md`, `backend/README.md`, `TECH_STACK.md`, 또는 `backend/docs/` 중 가장 가까운 문서를 같은 작업 브랜치에서 갱신한다.
- 문서 갱신이 필요 없다고 판단한 경우에는 최종 보고에 그 이유를 명시한다.
- 사용자-facing 문구, 샘플 데이터, 문서에서는 `개` 표현을 쓰지 않는다. 필요한 경우 `반려동물` 또는 더 중립적인 표현을 사용한다.

## 코딩 규칙

- 구현하지말고 패키지 최대한 사용
