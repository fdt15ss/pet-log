# 남은 페이지 작업

## 현재 상태

- repo root 기준 `frontend/app/web`에 Next.js App Router 앱이 구성되어 있습니다. `frontend` 디렉터리 안에서는 같은 위치를 `app/web`로 봅니다.
- mock API와 `localStorage` fallback을 사용하는 MVP 화면이 다음 경로에 구현되어 있습니다.
  - `/`
  - `/record`
  - `/analysis`
  - `/community`
  - `/more`
  - `/timeline`
  - `/suggestions`
  - `/profile`
  - `/notifications`
  - `/schedule`
  - `/settings`
  - `/shared-care`
  - `/hospital`
  - `/shopping`
- 하단 탭은 참고 이미지 기준인 `홈 / 기록 / 분석 / 커뮤니티 / 더보기` 구조로 맞춰져 있습니다.
- 기록 입력, 타임라인 날짜별 조회/검색/상세/수정/삭제, 프로필 편집과 사진 저장, 일정 저장, 알림 선호 설정, 커뮤니티 상세/댓글/글쓰기, AI 구조화 미리보기, 기록 기반 분석/제안이 동작합니다.
- 알림 화면은 기록 누락, 주의 기록 후속 확인, 저장된 일정 기반 리마인더를 보여주며 설정의 알림 선호를 반영합니다.
- 공동 관리, 병원 연계, 쇼핑 화면의 목업 입력과 선택 상태는 `localStorage`에 저장됩니다.
- 공동 관리, 병원 연계, 쇼핑 화면은 라우트 스모크 테스트와 빌드 검증을 통과했습니다.
- Next.js Route Handler 기반 mock API는 구현되어 있습니다. 실제 데이터베이스, 인증, 파일 업로드, 계정 간 동기화 기능은 아직 없습니다.

## MVP 완료 범위

### 1. 홈

- 반려동물 프로필 이름을 기준으로 오늘 화면을 표시합니다.
- 최근 기록과 AI 제안 진입 동선을 제공합니다.
- 기록이 없을 때의 빈 상태를 처리합니다.

### 2. 기록 입력

- 자연어 기록 입력과 카테고리 선택을 지원합니다.
- 입력 길이와 필수값을 검증합니다.
- 저장 전 AI 구조화 미리보기, 추천 분류, 신뢰도, 수치 추출 결과를 보여줍니다.
- 저장 직후 최근 기록과 타임라인에 반영합니다.

### 3. 타임라인

- 전체, 식사, 산책, 배변, 병원/약, 행동 필터가 실제 기록 목록과 연결되어 있습니다.
- 날짜별 기록 이동과 현재 날짜 기록 수, 주의/확인 요약을 제공합니다.
- 제목, 상세, 날짜, 시간, 카테고리, 구조화 요약, 수치 기준 검색을 지원합니다.
- 기록 카드를 눌러 상세 상태, AI 요약, 추출 수치를 펼쳐볼 수 있습니다.
- 기록 수정, 삭제, 삭제 확인, 수정 검증을 처리합니다.
- 필터 결과가 없을 때의 빈 상태를 제공합니다.

### 4. 분석과 제안

- 주간/월간 리포트 전환과 지표 필터를 제공합니다.
- 저장된 기록을 기준으로 누락 기록, 주의 기록, 안정 상태 인사이트를 생성합니다.
- 제안 화면은 기록 기반 AI 제안과 기본 제안을 함께 보여줍니다.
- AI 문구는 진단처럼 보이지 않도록 안전 안내를 포함합니다.

### 5. 프로필과 저장

- 반려동물 프로필 편집과 필수값 검증을 제공합니다.
- 프로필 사진 업로드와 모바일 촬영 입력을 지원하고 홈/프로필 화면 아바타에 반영합니다.
- 기록과 프로필은 `localStorage` 기반 상태로 새로고침 후에도 유지됩니다.
- 저장소 사용이 막힌 환경에서는 현재 세션 상태만 유지합니다.

### 6. 알림

- 홈과 상단 벨 버튼에서 오늘 확인할 알림으로 진입합니다.
- 저장된 기록을 기준으로 배변 기록 누락, 활동 기록 누락, 주의 기록 후속 확인을 생성합니다.
- 저장된 일정이 있으면 예방접종, 약 복용, 검진, 미용, 사료 변경 일정을 D-day 기준으로 알림에 반영합니다.
- 알림 읽음 상태를 저장하고 상단 벨 배지는 읽지 않은 알림 개수만 표시합니다.
- 알림 화면에서 개별 읽음 처리와 모두 읽음 처리를 제공합니다.

### 7. 일정

- 예방접종, 약 복용, 건강검진, 미용, 사료 변경 일정을 추가할 수 있습니다.
- 예정일, 반복 주기, 메모를 저장하고 완료 처리 또는 삭제할 수 있습니다.
- 3일 내 일정과 지연 일정을 요약하고 알림 화면과 상단 알림 카운트에 연결합니다.

### 8. 설정

- 더보기 화면에서 `/settings`로 이동할 수 있습니다.
- 기록 누락, 주의 기록 후속 확인, 일정 리마인더 알림을 각각 켜고 끌 수 있습니다.
- 알림 선호는 홈 알림, 알림 목록, 상단 알림 카운트에 반영됩니다.
- AI 요약과 케어 제안 토글을 제공하고 홈, 분석, 제안 화면의 AI 섹션 표시를 제어합니다.
- 프로필, 저장된 기록 수, 진행 중 일정 수를 요약하고 관련 화면으로 이동할 수 있습니다.
- 프로필, 기록, 일정, 설정을 JSON 파일로 내보낼 수 있습니다.
- 저장 데이터를 기본 예시 데이터로 되돌리는 초기화 확인 흐름을 제공합니다.

### 9. 커뮤니티

- 인기글, 최신글, 내 주변 피드와 게시판 필터를 제공합니다.
- 글 카드를 선택하면 본문, 작성자, 태그, 댓글을 상세로 확인할 수 있습니다.
- 댓글을 작성하면 선택된 글의 댓글 수와 댓글 목록에 즉시 반영됩니다.
- 글쓰기 폼에서 게시판, 제목, 내용을 입력해 최신글로 추가할 수 있습니다.
- 글 저장 토글을 제공하지만 계정 기반 저장은 아직 연결하지 않았습니다.

### 10. 공동 관리 / 병원 연계 / 쇼핑 UI

- 더보기 화면에서 공동 관리, 병원 연계, 쇼핑 화면으로 진입할 수 있습니다.
- 공동 관리는 보호자 초대 입력, 역할 선택, 저장된 초대 준비 목록, 활동 기록, 알림 공유 범위를 보여줍니다.
- 병원 연계는 저장되는 증상 및 변화 메모, 근처 동물병원 지도 목업, 병원 선택, 최근 주의 기록, 병원 제출용 리포트 미리보기, 방문 전 체크 항목을 보여줍니다.
- 쇼핑은 프로필과 기록 기반 추천 카드, 추천 이유, 저장되는 카테고리 필터와 추천 저장 상태, 구매 연결 제외 범위를 보여줍니다.
- 실제 초대 발송, 권한 검증, 지도 API 기반 주변 검색, 병원 예약 및 전송 API, 결제와 제휴 링크는 아직 연결하지 않았습니다.

## 홈 화면 UI 기획 대조

### 대조 기준

- 기준 문서: repo root 기준 `기획.md`, `frontend` 기준 `../기획.md`
- 기준 이미지: repo root 기준 `pet-log-ui.png`, `frontend` 기준 `../pet-log-ui.png`
- 제외 이미지: repo root 기준 `펫로그_20260428/*`, `frontend` 기준 `../펫로그_20260428/*`는 잘못된 예시이므로 홈 UI 판단에 사용하지 않습니다.

### 현재 충족 항목

- `기획.md`의 "AI가 먼저 질문하는 한줄 구간"은 홈 상단 AI 질문 카드로 반영되어 있습니다.
- 빠른 기록 버튼은 홈 상단 우측 `+` 버튼과 AI 질문 카드의 `기록하기` 버튼으로 제공합니다.
- 기록 누락 알림과 이상 징후 알림은 `오늘 알림` 섹션에서 일부 반영되어 있습니다.
- `오늘 요약`, `AI 제안`, `오늘 할 일`, `최근 기록` 섹션이 홈에 배치되어 있습니다.
- `pet-log-ui.png`의 모바일 대시보드 구조처럼 카드형 섹션과 하단 탭 구조를 유지하고 있습니다.

### 반영 완료 항목

- 홈 상단에 사진, 품종, 나이, 체중, 특이사항을 포함한 펫 프로필 요약 카드가 추가되었습니다.
- `오늘 요약`은 저장된 `records` 기반으로 식사, 산책, 배변 상태를 계산합니다.
- `최근 변화` 카드가 추가되어 주의 또는 확인 필요한 기록을 별도로 보여줍니다.
- `오늘 알림`은 알림 유형별 색상, 라벨, 우선순위를 사용합니다.
- 홈 첫 화면의 정보 위계는 `프로필 요약 -> AI 질문 -> 오늘 알림 -> 오늘 요약 -> 최근 변화 -> AI 제안 -> 오늘 할 일 -> 최근 기록` 순서로 정리되었습니다.

### 추가 후보

- 최근 기록 카드에 행동 해석 문구를 더 붙여 `최근 행동 요약` 성격을 강화할 수 있습니다.
- `pet-log-ui.png`의 촘촘한 대시보드 느낌은 유지하되, 320px 모바일에서 정보 밀도가 과하지 않은지 추가 시각 QA가 필요합니다.

## MVP 이후 확장 우선순위

### 1순위. 다음 작업 후보

- UI 완성도 QA: `pet-log-ui.png` 기준으로 전체 주요 화면의 모바일 줄바꿈, 버튼 높이, 카드 밀도를 추가 점검합니다.
- 커뮤니티 고도화: 글 상세를 별도 라우트로 분리하고 계정 기반 저장, 신고, 위치 권한 흐름을 추가합니다.
- 확장 UI QA 보강: 브라우저 자동화 도구가 준비되면 공동 관리, 병원 연계, 쇼핑의 모바일 스크린샷과 실제 클릭 저장 흐름을 추가 검증합니다.

### 2순위. 제품화 준비 후보

- 공동 관리 서버화: 실제 초대 발송, 계정별 권한 검증, 변경 이력 저장을 추가합니다.

- 알림 서버화: 반복 일정 자동 갱신, 실제 푸시, 브라우저 알림, 시간대 보정을 서버 작업으로 분리합니다.
- 데이터 백업/동기화: 계정 기반 저장, 기기 간 동기화, 복구 흐름을 추가합니다.

### 후순위. 확장 후보

- 실제 지도 API, IoT, 지출 관리는 사용자가 MVP 범위로 명시하기 전까지 후순위로 둡니다.

## 스프린트 계획

### 스프린트 1. 기록 입력과 세션 타임라인 연결

목표: 사용자가 `/record`에서 입력한 기록이 앱 안에서 실제 데이터처럼 이어지게 만듭니다.

상태: 완료

- `/record` 입력 상태와 저장 흐름을 구현합니다.
- 기록 입력값의 필수값, 길이, 카테고리 검증을 추가합니다.
- 저장된 기록을 세션 상태에 추가하고 `/timeline`에 반영합니다.
- 저장 직후 최근 기록 미리보기와 이동 흐름을 연결합니다.
- 기록이 없을 때의 빈 상태를 추가합니다.

완료 기준: 새로 입력한 기록이 새로고침 전 세션에서 홈 또는 타임라인에 표시되고, 잘못된 입력은 저장되지 않습니다.

결과: 공용 클라이언트 기록 상태를 추가했고, `/record`에서 저장한 내용이 홈 최근 기록과 `/timeline`에 즉시 반영됩니다.

### 스프린트 2. 필터, 선택 UI, 데이터 타입 안정화

목표: 현재 정적 UI를 실제 조작 가능한 MVP 화면으로 정리합니다.

상태: 완료

- 기록, 반려동물 프로필, 분석 지표, AI 제안, 커뮤니티 항목 타입을 분리합니다.
- 더미 데이터를 타입 기준에 맞춰 정리합니다.
- 타임라인, 분석, AI 제안, 커뮤니티 필터를 실제 목록 필터와 연결합니다.
- pill, segmented control, 탭 UI의 활성 상태를 구현합니다.
