# 디자인 스펙: AI 구조화 기록 미리보기 에이전트 통합

- **날짜**: 2026-05-12
- **상태**: 초안 (Draft)
- **주제**: 키워드 기반 기록 구조화 로직을 백엔드 AI 에이전트로 교체하고, 분석 데이터를 공통 컨텍스트에 통합.

## 1. 개요
현재의 기록 구조화 로직은 프론트엔드(`ai-insights.ts`)의 키워드 매칭에 의존하고 있습니다. 본 설계는 이를 강력한 백엔드 에이전트 기반 워크플로우로 대체합니다. "분석"과 "저장" 단계를 분리하여 사용자가 DB에 저장하기 전 AI가 생성한 후보를 검토하고 수정할 수 있도록 합니다. 또한, AI 기반 인사이트와 제안을 전역 `PetLogProvider`에 통합하여 페이지 간 일관된 경험을 제공합니다.

## 2. 아키텍처

### 2.1 백엔드 에이전트 (Backend Agents)
- **RecordStructuringAgent**: 자연어 입력을 분석하여 하나 이상의 `StructuredRecordCandidate`를 포함하는 `StructuredRecordBatch`를 반환합니다.
- **RiskDetectionAgent**: 최근 기록에서 잠재적인 건강 위험이나 이상 징후를 식별합니다.
- **SuggestionAgent**: 현재 맥락과 위험 신호를 바탕으로 케어 액션을 제안합니다.

### 2.2 프론트엔드 레이어 (Frontend Layers)
- **API 클라이언트 (`api-client.ts`)**:
  - `structureRecordPreview(detail, fallbackCategory)`: 저장 없이 후보군만 가져오기 위해 `source="ai_preview"` 옵션으로 백엔드를 호출합니다.
  - `fetchAiInsights(petId)`: 건강 인사이트를 조회합니다.
  - `fetchAiSuggestions(petId)`: 실행 가능한 케어 제안을 조회합니다.
- **PetLogProvider (컨텍스트)**:
  - 전역 상태 관리: `insights`, `suggestions`, `isAnalysisLoading`.
  - 갱신 로직 오케스트레이션: 기록 저장 성공 후, 인사이트와 제안을 가져오는 API를 병렬로 호출합니다.
- **기록 페이지 컴포넌트**:
  - **통합 요약 (방식 B)** 표시: 전체 요약을 보여주는 메인 카드와 검토를 위한 개별 기록 후보 리스트를 렌더링합니다.

## 3. 데이터 흐름 (Data Flow)

### 3.1 구조화 워크플로우 (미리보기 -> 검토 -> 확정)
1. **트리거**: 사용자가 기록 페이지에서 텍스트를 입력하거나 음성을 변환합니다.
2. **디바운스**: 250ms 지연 후 `structureRecordPreview` API가 호출됩니다.
3. **분석**: 백엔드 에이전트가 후보 묶음(Batch)을 반환합니다.
4. **미리보기**: 프론트엔드가 "통합 요약" 카드를 렌더링합니다. 사용자는 입력 내용에서 추출된 개별 항목(예: 식사, 산책)을 확인합니다.
5. **검토/수정**: 사용자는 미리보기 카드에서 카테고리나 수치를 수동으로 조정할 수 있습니다.
6. **확정(Commit)**: 사용자가 "저장"을 클릭하면, 프론트엔드는 **최종 확인된 후보 데이터**를 백엔드로 전송하여 저장합니다.

### 3.2 저장 후 분석 데이터 갱신
1. **성공**: 기록 저장(`addRecord`)이 성공합니다.
2. **갱신**: `PetLogProvider`가 `refreshAnalysis()`를 트리거합니다.
3. **병렬 조회**: `fetchAiInsights`와 `fetchAiSuggestions`가 동시에 호출됩니다.
4. **업데이트**: 홈, 분석, 제안 UI 컴포넌트가 최신 AI 콘텐츠로 리렌더링됩니다.

## 4. UI 디자인: 통합 요약 (방식 B)
- **헤더**: 상위 수준의 요약 텍스트 (예: "오늘 오전 케어 2건이 확인되었습니다").
- **본문**: 
  - 아이콘과 함께 감지된 카테고리 리스트 표시.
  - 추출된 수치들을 배지/태그 형태로 표시.
- **푸터**: 
  - "AI 추천 분류 적용" 버튼 (추천된 분류가 수동 선택과 다를 경우).
  - "전체 승인 및 저장" 버튼.

## 5. 예외 처리 및 성능
- **스켈레톤 상태**: 에이전트가 처리 중일 때 `isAnalysisLoading`을 사용하여 플레이스홀더를 표시합니다.
- **폴백(Fallback)**: 에이전트 분석이 실패할 경우, 원문 텍스트를 기반으로 한 단순 "수동" 표시 방식으로 돌아갑니다.
- **독립성**: 인사이트/제안 에이전트가 일시적으로 중단되더라도 기록 저장 기능은 정상적으로 작동해야 합니다.

## 6. 테스트 전략
- **단위 테스트**: `PetLogProvider`가 갱신을 올바르게 트리거하고 상태를 업데이트하는지 확인합니다.
- **통합 테스트**: 백엔드 에이전트 응답을 모킹하여 "통합 요약" 렌더링을 검증합니다.
- **E2E 테스트**: "입력 -> 미리보기 -> 수정 -> 저장 -> 홈 인사이트 확인" 전체 흐름을 테스트합니다.
